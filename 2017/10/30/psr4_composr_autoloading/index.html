composer autoload 解析

发表于 2017-10-30 | 分类于 php

概述

在阅读完psr-4 autoloader 标准以及实现完两个例子以后,对autolading机制已经非常了解.然后对我们经常使用的工具 composer 的autoload 机制进行一次解析.

composer autoloader

首先ComposerAutoloaderInit Class->getLoader 完成autolader的注册.类名后的随机字符串因该是防止重名(Guess)

getLoader

getLoader里的代码实现兼容了HHVM,PSR4,PSR0 等多种不同版本的autolad规范,此处我们只关心psr4相关的,毕竟psr0实例版本.HHVM暂时没接触。
getLoader做了以下操作
1.注册autoload 函数,插入在队列首部(只加载ClassLoader 文件)
2.实例化ClassLoader
3.取消autoload 函数
4.版本判断(HHVM,>5.6)
5.引入 autoload_static
6.通过call_user_func 完成 ClassLoader实例的属性绑定

autoload_static

autoload_static 文件存储了当前vendor目录下的package 的名空间和类文件对应关系.
prefixLengthsPsr4: 存放命名空间
prefixDirsPsr4: 命名空间对应DIR
在getInitializer 返回了一个作用域为 ClassLoader 的匿名方法;
值得注意的是prefixLengthsPsr4 以首字母进行分组.

classLoader

ClassLoader代码简单明了.
1.loadClass ->findFile
2.findFile->findFileWithExtension 完成命名空间对应的文件
3.includeFile 方法导入文件

classLoader代码过长,此处之列举核心方法;

# php # psr
psr-3
